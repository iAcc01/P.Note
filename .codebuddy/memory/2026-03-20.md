# 2026-03-20

## 架构改造：Electron + React 双端架构重构

将 PromptNote 从简单的 Electron+React 项目改造为支持 Web/macOS 双端的架构：

### 新增模块
- **`src/platform/context.tsx`** — PlatformProvider + usePlatform hook，自动检测 Web/Electron 环境
- **`src/services/promptService.ts`** — IPromptService 数据层抽象接口 + MockPromptService 实现
- **`src/services/index.ts`** — ServicesProvider + useServices/usePromptService hooks
- **`src/types/prompt.ts`** — 统一 Prompt 类型定义（从 mockData 解耦）
- **`src/router/routes.tsx`** — React Router 集中式路由配置 + 导航菜单配置
- **`src/router/index.ts`** — 路由模块统一导出

### 改造的文件
- **`main.tsx`** — 整合 PlatformProvider → ServicesProvider → HashRouter 三层嵌套
- **`App.tsx`** — 使用 useRoutes() 替代 useState + switch/case 手动路由
- **`Sidebar.tsx`** — 使用 React Router 的 useNavigate/useLocation 导航 + 平台感知
- **`StatusBar.tsx`** — 平台感知组件，仅桌面端显示
- **`PromptCard.tsx`** — 使用统一 Prompt 类型
- **`MyCreations.tsx`** — 通过 usePromptService() 获取数据

### 删除的文件
- `src/data/mockData.ts` — 数据迁移到 services/promptService.ts

### 新增依赖
- `react-router-dom`

### 使用 HashRouter 原因
兼容 Electron 的 file:// 协议，桌面端和 Web 端均可正常工作。
